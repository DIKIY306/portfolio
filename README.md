# Портфолио Дмитрия Лаврука

Современное интерактивное single-page портфолио веб-разработчика. Проект собран на чистых HTML, CSS и JavaScript без тяжёлых зависимостей.

## 1. Как открыть сайт

Самый простой способ: открыть файл `index.html` двойным кликом.

Для локального сервера:

```bash
cd portfolio_dmitry_lavruk_web
python3 -m http.server 4174
```

Затем открыть `http://localhost:4174`.

## 3. Как заменить сайты в визуальной подборке на свои кейсы

Карточки находятся в секции `#works` файла `index.html`. Детали модальных окон находятся в объекте `projects` файла `script.js`.

Для каждого реального кейса:

1. заменить название, категорию и описание карточки;
2. добавить свой скриншот или локальное изображение в `assets/`;
3. заменить задачу, список работ, результат и стек в `script.js`;
4. заменить ссылку кнопки `Открыть сайт` на адрес опубликованного проекта;
5. при использовании своих кейсов заменить подпись `Live website` на нужную метку.

## 4. Как подключить форму

Сейчас форма работает в demo-режиме: показывает уведомление и не отправляет данные.

Обработчик расположен в конце `script.js`:

```js
$("#contact-form").addEventListener("submit", (event) => {
  event.preventDefault();
  // Здесь можно отправить FormData в Telegram bot, email handler или CRM.
});
```

Варианты подключения:

- Formspree или Web3Forms для быстрого старта;
- serverless function на Netlify или Vercel;
- собственный backend;
- Telegram bot через защищённый backend. Не размещайте токен бота в клиентском JavaScript.

## 5. Как выложить сайт

### GitHub Pages

1. создать репозиторий;
2. загрузить содержимое папки проекта;
3. открыть `Settings → Pages`;
4. выбрать публикацию из ветки `main`.

### Netlify

Перетащить папку проекта в [Netlify Drop](https://app.netlify.com/drop) или подключить GitHub-репозиторий. Build-команда не нужна.

### Vercel

Импортировать GitHub-репозиторий в Vercel. Framework preset: `Other`. Build-команда не нужна.

## Визуальная подборка

В секции `#works` находятся шесть реальных сайтов с прямыми ссылками на оригиналы. Оптимизированные превью сохранены локально в `assets/`, чтобы GitHub Pages загружался быстро. Скриншоты выбраны из каталога [Referest](https://referest.ru/inspiration/web/pages).

Перед публикацией портфолио как списка выполненных заказов замените подборку собственными проектами и скриншотами.

## Структура

- `index.html` — контент и секции;
- `styles.css` — визуальная система, анимации и адаптив;
- `script.js` — меню, scroll reveal, фильтры, модальные окна и demo-форма;
- `assets/` — локальные изображения и SVG;
- `assets/dmitry-lavruk-avatar.jpg` — аватар Дмитрия;
- `CONTENT_TODO.md` — чек-лист данных для замены.
